Pure CSS Menu With One or More Submenus#UI-NA-001

This document serves four purposes:

  1. A (very) brief explanation of the menu.
  2. Steps to implement the menu into your code.
  3. Known issues.
  4. More information.

1. A (very) brief explanation of the menu.

The menu is pure CSS (no JavaScript). It relies on an ID in every <body> tag to work.

2. Steps to implement the menu into your code.

  1. Write your menu in valid XHTML, beginning with an unordered list <ul>.

    The only element that needs to have either a class or an ID is the root <ul>. I say 'root' because the menu could have zero or more submenus and therefore could contain more than one unordered list.

    You may use either a class or an ID as a hook into the menu, and you may name it anything you like. Note that the hashes in the href attributes are only there for reasons of simplicity and readibility. Also, none of the links have an ID attached to them (yet).

  2. Add a unique identifier to each link.

    Begin prepping the menu to work hand-in-glove with CSS by attaching an ID to each link. I prefer to use a descriptive word taken from the link text and prepend it to "Href", as I'll always know that that particular ID resides in the navigation menu.

    Each ID, in conjunction with an ID in the <body> tag, will act as a hook for the CSS to apply your styles to each link when selected. This will become clearer shortly.

  3. Add a unique identifier to each <body> element.

    This step is crucial to having a successful, working menu. While you're free to using any naming convention you'd like, I strongly recommend using one that is visibly similar to its "sister" ID in the navigational menu. For instance, I'd use the following for details.html:

    <body id="detailsPage">

    Remember, the ID for that particular link in the menu is #detailsHref. The similarities help me to get a good grasp of the behavorial aspect of the menu, even from just looking at the CSS. Later we'll see how using the two IDs together in a rule will highlight the selected link in the menu.

  4. Write the CSS rules for the menu.

    You can view all of the rules for the menu here.

    I'll quickly review the rules that influence the behavior of the menu. Note that these rules below may only be excerpts. Refer to the stylesheet itself to see the complete rules.

    1. Ensure that any submenus are closed by default (in above case, just one).
      ul#menu ul { display: none; }
    2. Open a submenu when appropriate.
      body#paymentsPage ul ul,
      body#historyPage ul ul,
      body#accountsPage ul ul,
      body#directPage ul ul,
      body#preferencesPage ul ul,
      body#postponePage ul ul { display: block; }

      For instance, if you need your submenu to open on a particular page, it MUST be included as a rule in the preceeding list. So, the submenu will be displayed when the user clicks on "Make Payments", it will be displayed when "History" is clicked on, etc. on down the list.

    3. Add the rules that will highlight the selected menu item.
      #indexPage a#indexHref,
      #loanDetailPage a#loanDetailHref,
      #paymentPage a#paymentHref,
      #historyPage a#paymentHref,
      #historyPage a#historyHref,
      #bankingPage a#paymentHref,
      #bankingPage a#bankingHref,
      #directPage a#paymentHref,
      #directPage a#directHref,
      #billPage a#paymentHref,
      #billPage a#billHref,
      #postponePage a#paymentHref,
      #postponePage a#postponeHref,
      #applicationsPage a#applicationsHref,
      #accountPage a#accountHref,
      #toolsPage a#toolsHref {
        color: #333;
        font-weight: bold;
        text-decoration: none;
      }
