Tooltip

This document serves four purposes:

  1. A (very) brief explanation of Tooltip.
  2. Steps to implement Tooltip into your code.
  3. Known issues.
  4. More information.

1. A (very) brief explanation of Tooltip.

Simply put, Tooltip is a popup contextual help box that can contain either plain text or HTML. Tooltip can be configured to be placed directly over the link or relative to where the mouse clicks on the target area. Note that Tooltip must have a parent element relatively positioned for it to consistently launch directly over the link. Tooltip began as an alternative to a large and unwieldy JavaScript library that was serving as the standard tooltip for sites designed by CEU.

2. Steps to implement Tooltip into your code.

  1. Include Tooltip in the link that's launching it.

    First, the anchor that is launching Tooltip must have the word 'tooltip' plus the tooltip number included as part of the href (i.e. tooltip23, tooltip0, tooltip5). Also, there could be a descriptive class name included within the anchor element (depending on how Tooltip should be displayed).

    <a href="lib/toolbox.html#tooltip3" class="foobar Tooltip_id-content eagles">My Super-cool Tooltip</a>

    Note that Tooltip may be included in an anchor with more than one class and that the order is unimportant.

    Tooltip can be set up in two different ways:

    I. Positioned relatively to its immediate parent, which will launch Tooltip directly over the beginning of the target area (this is the default behavior).

    - NOTE: If no class is given within the anchor, or if there are classes but none begin with "Tooltip_", no elements are relatively positioned via DOM scripting. This scenario is desirable when none of the parents are positioned.

    - To give the anchor an inline style that is relatively positioned (i.e. style="position: relative;") via DOM scripting. In other words, the script dynamically places the inline style into the anchor, so the coder doesn't need to.

    Tooltip_default

    - To relatively position an ID, i.e. document.getElementById("content"), include the following class within the anchor that's launching Tooltip:

    Tooltip_id-content

    - To relatively position an element(s) by its class, include the following class withing the anchor that's launching Tooltip:

    Tooltip_class.myClassName

    - To relatively position one or more node elements within a block element that has an ID, include the following class withing the anchor that's launching Tooltip:

    Tooltip_nav-li

    This example would be appropriate if there were a menu contained in an unordered list (<ul>) that had "nav" as its ID, and you wished to drill down into it and relatively position all of the list items (<li>).

    - To relatively position one or more classed node elements within a block element that has an ID, include the following class withing the anchor that's launching Tooltip:

    Tooltip_nav.myClassName

    This example would be appropriate if there were a menu contained in an unordered list (<ul>) that had "nav" as its ID, and you wished to drill down into it and relatively position all of the classed node elements with the class "myClassName".

    II. Launched relative to where the target area is clicked with the mouse.

    - To launch Tooltip respective to where the mouse is clicked within the target (link) area, include the following class withing the anchor that's launching Tooltip:

    Tooltip_mouse
  2. Create a separate HTML document that will house the tooltips.

    This file has a twofold purpose: first, it will be the container for each tooltip, wrapped in its own <div> that Ajax will grab and use as the tooltip, and second, it will be the page seen by users who either have JavaScript turned off or have an ancient browser incapable of supporting JavaScript. The latter is an example of graceful degradation.

    For this example I created another HTML document that I called "toolbox.html". Note that it doesn't matter where the file is placed in the web directory filesystem as long as it's world-readable. That means that the path contained in the link could be either absolute or relative, and that Tooltip can handle different tooltips in different files. In other words, there is no need to keep all tooltips in one file.

    In "lib/toolbox.html" I create my tooltips (or wherever you want). The following is an example of how the tip should look:

    <p><a name="tooltip3">My Super-cool Tooltip</a></p>
    <div id="tooltip3">
      <p>This space for rent.</p>
    </div><!--end div#tooltip3-->

    There are several things to be aware of:

    1. The ID of the <div> must say "tooltip" and then match the tooltip number from the link that's launching Tooltip (i.e. tooltip3). Also, if a class is added to the <div> it MUST appear after the ID. The only character allowed between the opening <div> and the id is a space (" ").
    2. The comment after the closing <div> tag MUST appear as seen, character for character, space for space.
    3. The anchor must be included above the tooltip within the HTML with the same characters that are in the <div> below it. This will serve those without the benefit of JavaScript, for they will be taken to this anchor after they click through the Tooltip link on the originating page. Note that the paragraph that serves as its parent element was arbitrarily picked and pressed into service. Any block element could be used here.
  3. Link to the external Javascript files from within any page that has Tooltip.
    <script type="text/javascript" src="my/path/to/tooltip.js"></script>
    <script type="text/javascript" src="my/path/to/coords.js"></script>
    <script type="text/javascript" src="my/path/to/xhr.js"></script>
  4. Link to the external CSS file from Toolbox.html and from within any page that has Tooltip.
    <link rel="stylesheet" type="text/css" href="my/path/to/tooltip.css" />
  5. Download all of the files.

3. Known Issues

  1. Tooltip may not appear directly over the link when one or more of its parent elements are absolutely or relatively positioned.

*Seems* to have been resolved by positioning an immediate parent element to relative. Also, choosing to launch Tooltip according to its mouse coordinates will work everytime. Tooltip ver.4, April 18, 2007
