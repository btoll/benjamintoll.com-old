Primitive types and reference values

In ECMAScript a variable can hold one or two types of values; primitive values and reference values.

  • Primitive values are simple pieces of data that are stored on the stack, which is to say that their value is stored directly in the location that the variable accesses.
  • Reference values, on the other hand, are objects that are stored in the heap, meaning that the value stored in the variable location is a pointer to a location in memory where the object is stored.

Primitive Types

ECMAScript has five primitive types: Undefined, Null, Boolean, Number and String. ECMA-262 defines the term type as a set of values, and each of the primitive types defines a range of values it can contain as well as literal representations of that type. To determine if that value is in the range of values for a particular type, ECMAScript provides the typeof operator. This operator can be used to determine if a value represents a primitive type and, if so, which primitive type it represents.

Reference Types

Reference types are commonly referred to as classes, which is to say that when you have a reference value, you are dealing with an object.

Let's look at some examples:


var x = 35;
alert(x.toFixed(2)); //returns "35.00";

var y = new Number(42);
alert(y.toFixed(2)); //retuns "42.00";

alert(typeof x); //returns "number" (data is stored on the stack);
alert(typeof y); //returns "object" (data is stored on the heap);

alert(x instanceof Number); //returns "false";
alert(y instanceof Number); //returns "true";
